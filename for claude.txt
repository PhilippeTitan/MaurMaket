SECTION 1: CRITICAL ISSUES (17) — FIX IMMEDIATELY
================================================================================

--- C1. Items undefined in cancel handler (server.js:2393) ---
Agents: Reliability, Security, Buyer/Seller
The cancel handler for paid orders references `items.rows` but `items` was never
queried. ReferenceError crashes the handler. Paid orders CANNOT be cancelled.
Fix: Add `const items = await client.query('SELECT product_id, quantity FROM order_items WHERE order_id = $1', [req.params.id]);` before line 2393.

--- C2. Cron auto-refund references order.rows[0] on plain object (server.js:5081) ---
Agents: Reliability, Security, Buyer/Seller
The cron job assigns `const order = orderResult.rows[0]` (a row object), then later
references `order.rows[0].total_amount`. TypeError crashes the cron. Auto-refund for
expired meetups NEVER fires. Buyers lose money.
Fix: Change `order.rows[0].total_amount` to `order.total_amount`.

--- C3. payout.completed uses MonCash reference as payout UUID (server.js:4155-4160) ---
Agent: Reliability
The handler does `UPDATE payouts SET status = 'completed' WHERE id = $1` with the
MonCash transaction reference (a string like tx_abc123), but payouts.id is a UUID.
Zero rows updated. Seller payouts stuck in "processing" forever.
Fix: Look up by moncash_reference instead of id.

--- C4. Duplicate orders on payment failure / back-button (CheckoutScreen.tsx:129-147) ---
Agent: Buyer/Seller
After createOrder succeeds, if createPayment fails, cart is NOT cleared. User can
re-checkout creating a duplicate pending order.
Fix: Always clear cart after createOrder succeeds (move clearCart outside the if block).

--- C5. No rate limiting on message sending (server.js:3663-3697) ---
Agent: Chat/Messaging
The message send endpoint has no dedicated rate limiter. A user can flood a
conversation with hundreds of messages per minute.
Fix: Add msgLimiter (30 messages/min per user).

--- C6. No rate limiting on conversation creation (server.js:3588-3622) ---
Agent: Chat/Messaging
No dedicated rate limiter. Malicious user can create hundreds of conversations.
Fix: Add conversationLimiter (10 conversations/min per user).

--- C7. imgbb API key leaked to all authenticated users (server.js:4986-4989) ---
Agent: Security
GET /api/upload/config returns the raw IMGBB_KEY to any authenticated user.
Fix: Remove this endpoint or proxy uploads server-side.

--- C8. MapTiler API key leaked publicly with no auth (server.js:4982-4984) ---
Agent: Security
GET /api/map-config returns the MapTiler API key with no authentication.
Fix: Remove the endpoint or add authRequired.

--- C9. Password change uses non-timing-safe SHA-256 fallback (server.js:836-837) ---
Agents: Security, Reliability
bcrypt fallback uses === for SHA-256 comparison. Timing attack vulnerability.
Fix: Use crypto.timingSafeEqual or remove the SHA-256 fallback entirely.

--- C10. Webhook stock decrement race condition (server.js:3994-4018) ---
Agent: Checkout/Payment
TOCTOU race: idempotency check happens before transaction. Two concurrent webhooks
can both pass the check, both decrement stock. Stock goes negative.
Fix: Check rowCount of UPDATE orders — if 0 rows updated, skip processing.

--- C11. N+1 Wishlist Checks in FeedScreen — 20 sequential API calls (FeedScreen.tsx:103-117) ---
Agent: Performance
Every feed load fires individual checkWishlist(p.id) API calls for all 20 products.
2-5 second network waterfall on slow connections.
Fix: Add batch endpoint POST /api/wishlist/check-batch.

--- C12. ExploreScreen calls Image.getSize() for every product (ExploreScreen.tsx:103-111) ---
Agent: Performance
After fetching 50 products, calls Image.getSize(url) for each image — 50 concurrent
HTTP requests to imgbb. Rate limiting, timeouts, battery drain.
Fix: Return image dimensions from backend at upload time.

--- C13. ExploreScreen masonry grid has no virtualization (ExploreScreen.tsx:326-345) ---
Agent: Performance
FlatList wraps entire masonry in single item — all 50 cards render at once.
2-4 second initial render, memory spike.
Fix: Use proper masonry library with virtualization.

--- C14. MeScreen uses ScrollView + .map() (MeScreen.tsx:297+533) ---
Agent: Performance
All seller products rendered at once in ScrollView. 50+ listings = 3-6 second render.
Fix: Replace with FlatList using ListHeaderComponent.

--- C15. FeedScreen renders same image twice per card (FeedScreen.tsx:288-291) ---
Agent: Performance
Each feed card renders the product image once blurred + once normal. 2x bandwidth.
Fix: Use a pre-generated blurred thumbnail instead.

--- C16. Order creation has N+1 product lookups (server.js:2236-2269) ---
Agent: Performance
For each cart item, runs 2 sequential DB queries (product + offer check).
5 items = 10 queries inside transaction holding locks.
Fix: Batch with WHERE id = ANY($1::uuid[]).

--- C17. Stock refund has broken total calculation (server.js:4041-4044) ---
Agent: Reliability
Dead code: `sum + (prodRow ? parseFloat(prodRow.quantity) * 0 : 0)` always sums to 0.
Fix: Remove lines 4041-4044, the actual refund amount on line 4047 is correct.

SECTION 2: HIGH ISSUES (43)
================================================================================

--- H1. Seller can advance unpaid orders to processing (server.js:3252) ---
Agent: Buyer/Seller
Transition map allows pending→processing. Seller ships items without payment.
Fix: Remove pending: 'processing' from transitions map.

--- H2. Review only attributed to first seller in multi-seller orders (server.js:1378-1383) ---
Agent: Buyer/Seller
LIMIT 1 on seller_id query. Other sellers in multi-seller orders never get reviewed.
Fix: Allow per-seller reviews (change UNIQUE constraint).

--- H3. "Complete" button missing for meetup orders (OrdersScreen.tsx:113) ---
Agent: Buyer/Seller
Button only shows for status==='delivered'. Meetup orders go paid→completed directly.
Fix: Show for paid + meetup delivery method.

--- H4. No message_type validation (server.js:3665) ---
Agent: Chat/Messaging
Arbitrary message_type accepted. Client can send any string.
Fix: Whitelist ['text', 'image', 'offer'].

--- H5. Conversation listing has no pagination + N+1 subqueries (server.js:3567) ---
Agent: Chat/Messaging
Returns ALL conversations with 2 correlated subqueries each. 500 convos = 1000 queries.
Fix: Add LIMIT/OFFSET, rewrite as JOINs.

--- H6. Read receipts mark ALL messages read on open (server.js:3636-3640) ---
Agent: Chat/Messaging
Opening conversation marks ALL unread messages as read, even unseen ones.
Fix: Pass lastReadMessageId from client.

--- H7. Image URL not validated on server (server.js:3678) ---
Agent: Chat/Messaging
imageUrl stored without validation. Potential SSRF/injection.
Fix: Validate https:// and allowed domains.

--- H8. Push notification avatar lookup never fires (server.js:3689) ---
Agent: Chat/Messaging
notifData missing 'type: new_message'. sendPushNotification checks data?.type, fails.
Fix: Add type: 'new_message' to notifData.

--- H9. Conversations list has 2 correlated subqueries per conversation (server.js:3569-3579) ---
Agent: Performance
30 conversations = 60 subqueries per request. 300ms+ added.
Fix: Denormalize with last_message_preview column.

--- H10. Seller analytics has 4 correlated subqueries (server.js:3378-3390) ---
Agent: Performance
4 scalar subqueries scan same data repeatedly.
Fix: Use CTEs or materialized view.

--- H11. Seller profile has 4 correlated subqueries (server.js:1693-1697) ---
Agent: Performance
4 subqueries per storefront visit.
Fix: Use JOINs or materialized view.

--- H12. Missing trigram index for search (server.js:1731-1734) ---
Agent: Performance
ILIKE '%term%' does full table scan. No pg_trgm index.
Fix: CREATE INDEX idx_products_name_trgm USING gin (name gin_trgm_ops);

--- H13. Product reviews endpoint has no pagination (server.js:1439-1455) ---
Agent: Performance
Returns ALL reviews with no LIMIT. Popular products = 50-100KB payload.
Fix: Add LIMIT/OFFSET.

--- H14. Wishlist endpoint uses correlated subquery per row (server.js:1476-1486) ---
Agent: Performance
30 wishlist items = 30 image subqueries. 150ms.
Fix: Batch-fetch primary images.

--- H15. Seller products endpoint uses correlated subquery for image (server.js:3190-3199) ---
Agent: Performance
50 products = 50 image subqueries. 250ms.
Fix: Batch with WHERE product_id = ANY($1::uuid[]).

--- H16. Product detail makes 5+ sequential API calls (ProductDetailScreen.tsx:59-125) ---
Agent: Performance
6 API calls in series. 1-3 seconds total load time.
Fix: Parallelize with Promise.all or single backend endpoint.

--- H17. No global body size limit on JSON parser (server.js:562-564) ---
Agent: Security
express.json() without explicit limit option.
Fix: Add { limit: '100kb' }.

--- H18. Morgan logs may capture Authorization headers (server.js:557) ---
Agent: Security
morgan('combined') logs full headers including Bearer tokens.
Fix: Use custom format that redacts Authorization header.

--- H19. Webhook logs full request body (server.js:3989) ---
Agent: Security
Logs full MonCash webhook payload including payment amounts and phone numbers.
Fix: Log only event type and reference.

--- H20. QR_SECRET falls back to JWT_SECRET derivative (server.js:2536) ---
Agent: Security
If QR_SECRET not set, QR tokens signed with JWT_SECRET + '_qr'.
Fix: Fail loudly if QR_SECRET not set.

--- H21. SSL certificate verification disabled for DB (server.js:24) ---
Agent: Security
ssl: { rejectUnauthorized: false }. MITM possible.
Fix: Use correct CA certificate.

--- H22. No rate limiting on expensive external API calls (server.js:4526) ---
Agent: Security
Verification submit calls 2 paid external APIs. Only general 100/min limiter.
Fix: Add per-user rate limit of 5 attempts/24h.

--- H23. Hardcoded Supabase URL reveals infrastructure (server.js:12+ locations) ---
Agent: Security
Supabase URL hardcoded as fallback in 12+ locations.
Fix: Use env vars exclusively.

--- H24. No security headers (missing helmet) (server.js:29) ---
Agent: Security
No X-Content-Type-Options, X-Frame-Options, HSTS, etc.
Fix: Install and use helmet middleware.

--- H25. CheckoutScreen order summary shows list price not sale price (CheckoutScreen.tsx:205) ---
Agent: Checkout/Payment
Line item shows item.price but subtotal uses effective_price. Numbers don't add up.
Fix: Use effective_price for line item display.

--- H26. retryPayment doesn't send returnUrl for deep-link (api.ts:283-284) ---
Agent: Checkout/Payment
Retry uses web URL fallback instead of maurmaket:// deep-link. Users stranded on web page.
Fix: Send deep-link returnUrl in retryPayment body.

--- H27. Promo validation holds FOR UPDATE lock unnecessarily (server.js:3289) ---
Agents: Buyer/Seller, Checkout/Payment
Read-only endpoint acquires row lock. Blocks other users validating same promo.
Fix: Remove FOR UPDATE from validate endpoint.

--- H28. Promo discount cap formula confusing (server.js:2284-2285) ---
Agent: Checkout/Payment
10% promo caps at Rs 100 max discount (discount_value * 10). Undocumented.
Fix: Remove cap or document clearly in UI.

--- H29. Multiple refund paths use deprecated payout API (server.js:2413,3051,4051,5086) ---
Agent: Checkout/Payment
All refunds use deprecated external-payout-create endpoint.
Fix: Migrate to payout-create, update field names.

--- H30. Subscription webhook may not extract order reference (server.js:4850-4851) ---
Agent: Checkout/Payment
Uses data?.referenceId but MonCash sends reference at top level. Subscription never activates.
Fix: Fall back to req.body.reference.

--- H31. Escrow endpoint leaks commission info to buyers (server.js:3098-3114) ---
Agent: Checkout/Payment
Commission_amount and net_amount visible to buyers. Sensitive business info.
Fix: Filter response for buyers.

--- H32. No connection release in webhook stock-refund path (server.js:4034-4066) ---
Agent: Reliability
Early return in ROLLBACK path skips client.release(). Connection leak.
Fix: Wrap in try/finally.

--- H33. payout.completed handler has no transaction or idempotency (server.js:4155-4160) ---
Agent: Reliability
Bare UPDATE without transaction. Race condition with payout.failed.
Fix: Wrap in transaction with FOR UPDATE.

--- H34. payout.failed can double-refund balance (server.js:4161-4194) ---
Agent: Reliability
Different event IDs for same payout could both trigger balance refund.
Fix: Ensure atomic status update + balance refund.

--- H35. Offer response not in transaction (server.js:3802-3866) ---
Agent: Reliability
Offer status updated, then message inserted. Message fails = inconsistent state.
Fix: Wrap in transaction.

--- H36. Conversation creation not transactional (server.js:3588-3622) ---
Agent: Reliability
Duplicate check + insert without transaction. Race condition creates duplicates.
Fix: Use INSERT ... ON CONFLICT or transaction.

--- H37. Image messages fixed 200x200 no aspect ratio (ChatScreen.tsx:441) ---
Agent: Chat/Messaging
All image messages render as 200x200 square. Wide/tall images distorted.
Fix: Capture dimensions at upload, render proportionally.

--- H38. Content length validation runs for all types including offer (server.js:3667-3668) ---
Agent: Chat/Messaging
5000-char limit checked for offer messages where content is irrelevant.
Fix: Only validate for text and image types.

--- H39. since parameter not validated as timestamp (server.js:3644) ---
Agent: Chat/Messaging
Garbage input like "oops" causes PostgreSQL error.
Fix: Validate with new Date() before query.

--- H40. N+1 correlated subqueries in conversation listing (server.js:3573-3574) ---
Agent: Chat/Messaging
Per-row subqueries for last_message and unread_count.
Fix: Rewrite as LEFT JOIN LATERAL.

--- H41. Notification cleanup only runs on startup (server.js:5168) ---
Agent: Chat/Messaging
Old notifications accumulate if server runs for weeks without restart.
Fix: Add cron job: daily at 3 AM.

--- H42. Offer respond missing conversation membership check (server.js:3802) ---
Agent: Chat/Messaging
Seller can respond to offers from any conversation if same seller_id.
Fix: Verify conversation membership.

--- H43. No server-side message deduplication (server.js:3663) ---
Agent: Chat/Messaging
Double-tap can send duplicate messages. No idempotency key.
Fix: Client-generated UUID as idempotency key.
